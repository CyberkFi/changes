# BST-001: Chuẩn hoá Scope Template & Quy trình Kỹ thuật cho Dịch vụ Tái cấu trúc

## Mục tiêu

Xây dựng scope template, quy trình kỹ thuật, và tài liệu hướng dẫn phục vụ cho 3 dịch vụ đã tái cấu trúc (Product Design, Product Build, Tech Labs). Đây là nền tảng để DEV và DES deliver đúng chuẩn.

## Bối cảnh

Cyberk đang chuyển từ ad-hoc pricing sang fixed-price model. BST cần định nghĩa rõ ràng scope, ranh giới, và quy trình kỹ thuật cho từng gói dịch vụ. Nếu thiếu scope template → Product Build $29K không biết bao gồm gì, dẫn đến scope creep.

## Việc cần làm

### 1. Scope Template cho Product Build

- [ ] Định nghĩa scope $29K (Basic): bao nhiêu screens, smart contracts, API endpoints, integrations
- [ ] Định nghĩa scope $35K (Pro): thêm gì so với Basic (CTO advisory, security audit, reports)
- [ ] Ranh giới rõ ràng: "IN scope" vs "OUT scope" cho mỗi tier
- [ ] Template add-on pricing: cách estimate ~$5K/feature thêm
- [ ] Ví dụ scope mẫu: dự án DeFi swap $29K vs dự án RWA tokenization custom quote

### 2. Scope Template cho Product Design

- [ ] Scope Basic ($5K): số lượng screens, responsive variants, design system components
- [ ] Scope Professional ($7K): Basic + landing page spec + research report outline
- [ ] Scope AIO ($8K): Pro + content deliverables list (số bài, video, SEO checklist)
- [ ] Revision policy: bao nhiêu vòng design revision included

### 3. Architecture & Infra Templates

- [ ] Project boilerplate template: repo structure, CI/CD, deploy config
- [ ] Monitoring setup template: Cách setup 15/30 ngày monitoring cho Product Build
- [ ] Landing page code template: reusable cho Product Design Professional

### 4. Quy trình Delivery

- [ ] Product Build 30-day sprint flow: Week 1 (setup) → Week 2-3 (dev) → Week 4 (QA+deploy)
- [ ] Product Design delivery flow: Kickoff → Research → Design → Review → Handoff
- [ ] Handoff checklist: design → dev (internal), product → client (external)
- [ ] Requirement change process: khi nào add-on, khi nào chuyển Tech Labs

### 5. Tài liệu Kỹ thuật

- [ ] Cập nhật Cyberk Framework documentation cho context dịch vụ mới
- [ ] Tech stack recommendation guide: khi nào dùng gì (Next.js vs React Native, Solidity vs Rust...)
- [ ] Security checklist cho Product Build Pro (đi kèm security audit deliverable)

## Phối hợp

- Phối hợp **DEV** — Validate scope templates có khả thi trong 30 ngày không
- Phối hợp **DES** — Validate scope Product Design phù hợp với năng lực team
- Phối hợp **QA** — Align QA checklist với scope templates
- Phối hợp **CEO** — Approve scope templates trước khi áp dụng cho client
